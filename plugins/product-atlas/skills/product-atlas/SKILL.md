## PRIMACY ZONE — Identity and Purpose

**What this atlas contains**

Product-atlas orchestrates product management, strategy, design, marketing, and launch planning skills.

**Use this atlas when**

- Developing product strategy
- Managing product development
- Designing product features
- Planning product launches
- Creating product marketing content
- Analyzing product-market fit

**Do not use this atlas when**

- Technical implementation (use frontend-atlas, backend-atlas)
- Pure marketing content (use content-atlas)
- Technical documentation (use docs-architect)

### Data Extraction for Product Management

**Use data-extraction for product research and competitive intelligence:**

| Product Activity | Data Source | Extraction Method |
|------------------|-------------|-------------------|
| **Competitor Analysis** | ProductHunt launches, G2/Capterra reviews, Amazon products | data-extraction/business-productivity + ecommerce-shopping |
| **User Feedback** | Reddit discussions, Medium articles, Quora questions | data-extraction/social-media-content |
| **Market Trends** | GitHub trending, Stack Overflow tags, Hacker News | data-extraction/developer-platforms |
| **Pricing Intelligence** | Amazon/eBay pricing, competitor products, e-commerce data | data-extraction/ecommerce-shopping |
| **Feature Research** | GitHub issues, Stack Overflow questions, user discussions | data-extraction/developer-platforms + social-media-content |
| **Launch Analysis** | ProductHunt comments, social media reactions, reviews | data-extraction/business-productivity + social-media-content |

**When to use:**
- **Competitive research** → Extract competitor products, reviews, pricing from G2, Capterra, ProductHunt
- **User needs** → Gather feedback from Reddit, Quora, social media discussions
- **Feature validation** → Research GitHub issues, Stack Overflow questions for pain points
- **Market sizing** → Analyze trends from GitHub, Stack Overflow, developer platforms
- **Launch planning** → Study successful launches on ProductHunt, social media

**Example workflows:**
```
Product Research: "Project management SaaS"
1. data-extraction/business-productivity/g2 → Extract competitor reviews
2. data-extraction/business-productivity/producthunt → Analyze launches
3. data-extraction/social-media-content/reddit → User pain points
4. Synthesize into product requirements

Feature Validation: "Add real-time collaboration"
1. data-extraction/developer-platforms/github → Research implementations
2. data-extraction/developer-platforms/stackoverflow → Common challenges
3. data-extraction/social-media-content → User expectations
4. Define feature scope based on research
```
